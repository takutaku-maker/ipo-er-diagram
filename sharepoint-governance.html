SharePoint統合 ガバナンスガイド

Revise株式会社のSharePoint統合（既存11サイト→新5サイト）における構成設計・運用ルールの基準文書

最上位方針：2層の優先順位

第1層 IPO・ISMSで最低限必要な構成

判断基準：「これがないと監査で指摘される / ISMS認証が取れない」

複雑化・煩雑化を避ける。含まれるもの：権限分離、職務分掌、文書管理、アクセス制御、監査証跡

第2層 業務上必要なもの

判断基準：「業務を回すために必要 / SSOTとして正本管理すべき」

含まれるもの：プロセスから発生する成果物、SSOTにつながる資料

最終形態：システム制御ワークフロー（Asana等）から自動生成されるアウトプットの保存先。ワークフロー経由でないファイル追加は例外扱い

判断ルール：「あったほうがいい」でフォルダを増やさない。第1層か第2層のどちらかの基準を満たすかで判断する。

5つの設計判断

判断 #1

SW資産の開発資料と資産性担保資料の分離

課題：開発資料と資産性担保資料を同一場所に置くべきか

結論：分けるべき。開発資料→02_業務共通、資産性担保資料→03_管理部門。「作る人」と「評価する人」を物理分離

根拠：J-SOX実施基準II.1「職務の分掌」/ 企業会計原則注解20 / ISMS A.8.4

判断 #2

規程はIPO推進フォルダに置かない

課題：IPO準備で整備する規程をIPO推進フォルダに配置すべきか

結論：置かない。規程は会社の恒久資産。全社規程原本→03法務総務、IT統制規程→04、公開版→05（読取専用）

根拠：ISMS箇条7.5.2/7.5.3 / J-SOX実施基準II.1

判断 #3

J-SOX 3点セットは管理部門サイトに一元管理

課題：業務記述書・フローチャート・RCMをどこに配置するか

結論：03_管理部門/06_内部統制に業務プロセス単位で格納。IT統制RCMは04と相互参照するが原本は03

根拠：金商法第24条の4の4 / 実施基準III.4.(2) / 監査実務委員会報告82号

判断 #4

サイト所有者にはIT管理責任者ではなく業務責任者を置く

課題：5サイトの管理者をどう決めるか

結論：01,03=管理部長、02,04=IT管理責任者、05=総務担当。サイトコレクション管理者は全社2名限定

根拠：J-SOX実施基準II.1 / ISMS箇条5.3 / A.8.2

判断 #5

パートナー別フォルダの再構成

課題：「41_ヤマダデンキ」「63_インターゾーン」等の個別番号付きフォルダをどうするか

結論：番号体系廃止。営業推進部/02_案件管理に統合。契約書は03_契約書管理に集約。プロセスのアウトプットのみ移行

根拠：J-SOX取引証憑管理 / ISMS A.5.20

多層SSOTアーキテクチャ（5層モデル）

SSOTは単一DBではなく多層で捉える基盤。SharePointの2層はフォルダ配置の軸、以下はデータの意味と役割の階層。
